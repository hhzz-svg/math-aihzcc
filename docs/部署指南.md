# 部署指南

## 一、部署结构

- 前端：Cloudflare Pages，项目名 `math-aihzcc`，发布目录 `public`。
- 自定义域名：`math.aihzcc.top`。
- 后端：Render Web Service，Node 运行时，启动 `npm start`。
- 源码：GitHub 仓库，不包含 `private/reference/*.pdf`。

## 二、Cloudflare Pages

项目根目录执行：

```powershell
npx wrangler@4.111.0 pages deploy public --project-name math-aihzcc --branch main
```

当前项目地址：

```text
https://math-aihzcc.pages.dev
```

自定义域名需要在 `aihzcc.top` 区域添加：

```text
Type: CNAME
Name: math
Target: math-aihzcc.pages.dev
Proxy: Proxied
```

Cloudflare Pages 项目中已经注册 `math.aihzcc.top` 后，DNS 生效会自动完成域名验证和证书签发。

## 三、Render

仓库包含 `render.yaml`，也可用 Render CLI：

```powershell
render services create `
  --name math-aihzcc-api `
  --type web_service `
  --repo https://github.com/hhzz-svg/math-aihzcc `
  --branch main `
  --runtime node `
  --build-command "npm install --omit=dev" `
  --start-command "npm start" `
  --health-check-path /api/health `
  --plan free `
  --region oregon `
  --output json
```

当前 Render 服务：

```text
https://math-aihzcc-api.onrender.com
https://math-aihzcc-api.onrender.com/api/health
```

前端的 `public/config.js` 已配置：

```js
window.MATH_SITE_CONFIG = {
  apiBase: "https://math-aihzcc-api.onrender.com",
  referenceEndpoint: "/local-reference/guide.pdf"
};
```

每次修改配置或正文后，重新部署 Cloudflare Pages。

## 四、PDF

### 本地

把个人参考书放到：

```text
private/reference/全国大学生数学竞赛参赛指南.pdf
```

`npm start` 后，网页会自动读取。

### 公开站点

不把 PDF 提交到 GitHub，也不放进 `public`。用户在原书阅读器里点击“选择本机 PDF”。

如配置私有 R2：

1. R2 bucket 保持私有；
2. 使用 Cloudflare Access 或短时签名 URL；
3. 保留 Range 支持；
4. 不在前端写入永久密钥；
5. 把受保护的 URL 配置在服务端环境变量，而不是仓库。

## 五、发布前检查

```powershell
npm test
curl https://math-aihzcc-api.onrender.com/api/health
curl -I https://math-aihzcc.pages.dev
curl -I https://math.aihzcc.top
```

检查：

- 首页、章节、计划、真题和资料页均可打开；
- KaTeX 无报错；
- Render API 返回 `ok: true`；
- 公开站点没有上传 `private/reference`；
- 自定义域名 HTTPS 正常。

## 七、章节原书摘页与学习计划字段（2026-07-16）

章节正文会直接展示 `public/assets/reference/pages/` 中的 PDF 摘页图片。当前摘页覆盖考纲相关物理页 26–29、历届试题入口页 92 和参考解答入口页 105；点击图片仍会进入原书阅读器。摘页图片是公开部署的轻量资源，完整个人 PDF 继续只在本地或 Render 私有参考目录中提供，不提交到 GitHub。

`data-override.js` 作为正文数据覆盖层加载在 `book-data.js` 之后，当前每章提供 8 个知识小节、1 道例题和 8 道练习；每周计划提供 `daily`、`quota`、`output`、`check` 与 `fallback` 字段。首页剩余时间由浏览器每秒刷新，切换到其他路由时会停止计时器。

公开站点如果显示 `host error`，先分别访问以下地址确认是哪一层：

- 前端：`https://math.aihzcc.top/`
- Pages 备用地址：`https://math-aihzcc.pages.dev/`
- 后端健康检查：`https://math-aihzcc-api.onrender.com/api/health`

若后端健康检查短暂失败，通常是 Render 免费实例冷启动；刷新后端地址再重新打开前端即可。前端正文、摘页图片和本地题库不依赖后端，后端恢复前仍可阅读和刷题。
